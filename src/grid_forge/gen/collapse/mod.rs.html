1//! # Collapsible procedural generation
2//!
3//! Algorithms contained within this module works by having a collection of tiles, which final identity is unknown in the beginning -
4//! *every position can be any tile*. In the process of generating a new grid, algorithm will *collapse* possible options for position,
5//! choosing randomly one of them. Collapsed tile will then put contraints over the remaining tiles, reducing the number of possible options.
6//! Process will then continue until all tiles are collapsed into one possible option.
7//!
8//! This process is called elsewhere most often *Model Synthesis* or *Wave Function Collapse*, though the distinction between the two is
9//! not always clear. Implementation contained there can be tailored to specific needs for the generation process, which makes the line
10//! even more blurry, so new name is used here.
11//!
12//! ## Main distinction
13//!
14//! Two main types of algorithms are provided in distinct submodules:
15//! - [`singular`] - implementation that can be used to generate maps. Each tile type needs to be self-descriptive in
16//!   regards to its possible neighbours. Constraints for this type of generation is based strictly on possible neighbours for each given tile,
17//!   set of rules which can be described as: tile `X` can be placed in direction `D` of tile `Y`.
18//! - [`overlap`] - implementation that can be used to generate maps based on observed patterns. Direct possible adjacents of each tile are
19//!   less important there, the pattern context is more important. First step for this type of generation is to create a collection of patterns
20//!   from sample gridmaps, which then will be tested for possible neigbours. During the process the possible patterns will then be collapsed
21//!   instead of individual tiles.
22//!
23//! ## Struct types
24//!
25//! Structs contained in submodules mentioned above are categorized in analogous way, with some additional distinctions described further
26//! in documentation for concrete struct.
27//!
28//! - *adjacency rules* are the rules that are used to determine possible neighbours for each tile.
29//! - *frequency hints* can be described as a weights per option - they can be derived from sample gridmaps and are a way to influence the
30//!   frequency of options occuring in the generated grid.
31//! - *analyzers* provide methods for analyzing sample gridmaps and creating rulesets out of them.
32//! - *collapsible grids* are the source of information for the *resolvers*, from the collection of collapsible tiles to the prepared
33//!   adjacency rules and frequency hints. They can also contain some pre-collapsed tiles, providing initial constraints for the generation.
34//! - *resolvers* are the main executors of the algorithm and are responsible for collapsing the tiles in the *collapsible grids*.
35//! - *queues* are used to determine the order in which tiles are collapsed: [`PositionQueue`] takes next position to collapse in a fixed
36//!   order, while [`EntrophyQueue`] fetch the next position to collapse with the lowest entrophy.

56#[derive(Clone, Debug, Default)]
57pub(crate) struct Adjacencies {
58    inner: Vec<HashSet<u64>>,
59}
60
61impl Adjacencies {
62    pub fn new() -> Self {
63        let mut inner = Vec::new();
64
65        for _ in 0..GridDir::ALL_2D.len() {
66            inner.push(HashSet::default());
67        }
68
69        Self { inner }
70    }
71
72    #[inline(always)]
73    pub fn add_at_dir(&mut self, direction: GridDir, id: u64) {
74        let v = self.inner.get_mut(direction as usize).unwrap();
75        v.insert(id);
76    }
77}
78
79impl Index<GridDir> for Adjacencies {
80    type Output = HashSet<u64>;
81
82    fn index(&self, index: GridDir) -> &Self::Output {
83        &self.inner[index as usize]
84    }
85}

87/// Basic Subscriber for debugging purposes.
88///
89/// Implements both [`overlap::Subscriber`] and [`singular::Subscriber`], making it usable with both resolvers.
90/// Upon collapsing a tile, it will print the collapsed `GridPosition`, `tile_type_id` and (if applicable) `pattern_id`.
91#[derive(Debug, Default)]
92pub struct DebugSubscriber {
93    file: Option<File>,
94}
95
96impl DebugSubscriber {
97    pub fn new(file: Option<File>) -> Self {
98        Self { file }
99    }
100}
101
102impl singular::Subscriber for DebugSubscriber {
103    fn on_collapse(&mut self, position: &GridPosition, tile_type_id: u64) {
104        if let Some(file) = &mut self.file {
105            writeln!(
106                file,
107                "collapsed tile_type_id: {tile_type_id} on position: {position:?}"
108            )
109            .unwrap();
110        } else {
111            println!("collapsed tile_type_id: {tile_type_id} on position: {position:?}");
112        }
113    }
114
115    fn as_any(&self) -> &dyn std::any::Any {
116        self
117    }
118}
119
120impl overlap::Subscriber for DebugSubscriber {
121    fn on_collapse(&mut self, position: &GridPosition, tile_type_id: u64, pattern_id: u64) {
122        if let Some(file) = &mut self.file {
123            writeln!(
124                file,
125                "collapsed tile_type_id: {tile_type_id}, pattern_id: {pattern_id} on position: {position:?}"
126            )
127            .unwrap();
128        } else {
129            println!(
130                "collapsed tile_type_id: {tile_type_id}, pattern_id: {pattern_id} on position: {position:?}"
131            );
132        }
133    }
134}

136pub(crate) mod private {
137    use std::collections::HashMap;
138
139    use crate::map::GridDir;
140
141    use super::Adjacencies;
142
143    #[derive(Clone, Debug, Default)]
144    pub struct AdjacencyTable {
145        inner: HashMap<u64, Adjacencies>,
146    }
147
148    impl AsRef<HashMap<u64, Adjacencies>> for AdjacencyTable {
149        fn as_ref(&self) -> &HashMap<u64, Adjacencies> {
150            &self.inner
151        }
152    }
153
154    impl AdjacencyTable {
155        pub(crate) fn insert_adjacency(&mut self, el_id: u64, direction: GridDir, adj_id: u64) {
156            match self.inner.entry(el_id) {
157                std::collections::hash_map::Entry::Occupied(mut e) => {
158                    e.get_mut().add_at_dir(direction, adj_id)
159                }
160                std::collections::hash_map::Entry::Vacant(e) => {
161                    let mut adjacencies = Adjacencies::new();
162                    adjacencies.add_at_dir(direction, adj_id);
163                    e.insert(adjacencies);
164                }
165            }
166        }
167
168        pub(crate) fn get_all_adjacencies_in_direction(
169            &self,
170            el_id: &u64,
171            direction: &GridDir,
172        ) -> impl Iterator<Item = &u64> {
173            self.inner
174                .get(el_id)
175                .expect("cannot get adjacencies for provided `el_id`")[*direction]
176                .iter()
177        }
178    }
