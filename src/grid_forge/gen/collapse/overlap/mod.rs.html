1//! # Overlapping collapsible generation
2//!
3//! Collapsible generation algorithm working with rulesets that are based on patterns of tile type patterns. Often named
4//! within other sources as *overlapping Wave-Function Collapse*.
5//!
6//! As it is based on multi-tile patterns, it is useful for generation where tiles itself don't have a clear adjacencies
7//! relation, but instead are grouped in meaningful patterns.
8//!
9//! ## Struct types
10//!
11//! In general the types are described in the documentation for [`collapse`](crate::gen::collapse) module.
12//!
13//! - [`AdjacencyRules`] and [`FrequencyHints`] are self-descriptive, and both can be created by the analyzer.
14//! - [`Analyzer`] can generate [`AdjacencyRules`], [`FrequencyHints`] and [`PatternCollection`].
15//! - [`PatternCollection`] is a collection of [`OverlappingPattern`]s gathered from sample maps. It is an additional
16//!   element over [`singular`](crate::gen::collapse::singular) workflow needed for translation between collapsed patterns
17//!   and underlying individual `tile_type_id`.
18//! - [`CollapsiblePatternGrid`] is the collection of [`CollapsiblePattern`], and is used by [`Resolver`] to generate
19//!   new map.
